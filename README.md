# Scenar Editor

Scenar Editor — это frontend-приложение на React для создания и редактирования интерактивных сценариев.

## Возможности

- просмотр и редактирование информации о сценарии;
- управление шагами сценария;
- создание шагов типов `текст`, `выбор`, `конец`;
- создание и редактирование вариантов выбора;
- настройка переходов между шагами;
- поиск по шагам;
- запуск проекта локально или через Docker.

## Установка и запуск локально

### 1. Клонировать репозиторий

```bash
git clone <ссылка-на-репозиторий>
cd scenar-editor
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Запустить проект

```bash
npm run dev
```

### 4. Открыть в браузере

После запуска приложение будет доступно по адресу:

```text
http://localhost:5173
```

---

## Запуск через Docker

### 1. Собрать Docker-образ

```bash
docker build -t scenar-editor .
```

### 2. Запустить контейнер

```bash
docker run --rm -p 5173:5173 scenar-editor
```

### 3. Открыть в браузере

```text
http://localhost:5173
```

## Технические требования к странице
### 13. Конструктор / редактор сценариев

Конструктор сценариев предназначен для создания и редактирования сценария.  
Страница конструктора состоит из следующих основных частей:

- `Окно шага`
- `Окно выбора`
- `Менеджер шагов`

В верхней части страницы располагаются общие элементы управления сценарием:

- **Кнопка** `Сохранить изменения`
- **Кнопка** `Редактировать информацию о сценарии`

При нажатии на кнопку `Редактировать информацию о сценарии` открывается всплывающее окно, содержащее:

- **Поля:** название сценария, описание сценария, preview сценария, тип доступа сценария, режим сценария
- **Кнопки:** `Сохранить изменения`, `Закрыть`

---

#### 13.1. Менеджер шагов

Менеджер шагов предназначен для навигации по шагам сценария и создания новых шагов.

- При создании сценария в нем уже существует один шаг — `начальный`
- Шаги отображаются в виде списка
- В менеджере шагов шаги располагаются в порядке их создания
- Для каждого шага в списке отображаются:
  - заголовок шага
  - тип шага
- При нажатии на шаг в списке в `Окне шага` открывается информация о выбранном шаге
- В менеджере шагов доступна **кнопка** `Добавить шаг`
- При нажатии на кнопку `Добавить шаг` создается новый шаг
- В менеджере шагов доступно **поле поиска** по названию шага

---

#### 13.2. Окно шага

Окно шага предназначено для просмотра и редактирования выбранного шага сценария.

В окне шага отображаются и могут редактироваться следующие поля:

- заголовок шага
- тип шага
- описание шага
- изображение шага

Доступные типы шага:

- `текст`
- `выбор`
- `конец`

Особенности редактирования шагов:

- начальный шаг нельзя удалить
- тип начального шага нельзя изменить

Для шага типа `текст` дополнительно отображается поле:

- `Следующий шаг`

Для шага типа `конец` поле выбора следующего шага отсутствует.

В окне шага доступны следующие кнопки:

- `Сохранить изменения`
- `Удалить шаг`

При нажатии на кнопку `Удалить шаг` открывается окно подтверждения удаления.

---

#### 13.3. Выборы шага типа `выбор`

Если выбран шаг типа `выбор`, в `Окне шага` дополнительно отображается список выборов данного шага.

Для каждого выбора отображаются:

- номер выбора
- укороченный текст выбора
- следующий шаг, к которому ведет выбор, если он указан
- изменение счета пользователя, если сценарий является оцениваемым и значение указано

Также отображается **кнопка** `Добавить выбор`.

Ограничения:

- у одного шага типа `выбор` может быть не более 4 выборов

При нажатии на карточку выбора данный выбор становится активным и открывается в `Окне выбора` для редактирования.

Активный выбор должен быть визуально выделен.

---

#### 13.4. Окно выбора

Окно выбора предназначено для редактирования активного выбора шага типа `выбор`.

В окне выбора отображаются:

- заголовок вида `Выбор #<номер>`
- название шага, к которому относится данный выбор

В окне выбора могут редактироваться следующие поля:

- текст выбора
- следующий шаг
- пояснение выбора

Если сценарий находится в режиме `оцениваемый`, дополнительно отображается поле:

- изменение счета пользователя

Если сценарий находится в режиме `неоцениваемый`, поле изменения счета пользователя не отображается.

В окне выбора доступны следующие кнопки:

- `Удалить выбор`
- `Обратно к шагу`

Кнопка `Сохранить изменения` в окне выбора отсутствует.

Изменения в полях окна выбора должны применяться сразу к локальному состоянию редактируемого сценария.

При нажатии на кнопку `Удалить выбор` выбор удаляется из текущего шага.

---

#### 13.5. Всплывающее окно редактирования информации о сценарии

Всплывающее окно редактирования информации о сценарии предназначено для изменения основных свойств сценария.

Во всплывающем окне могут редактироваться следующие поля:

- название сценария
- описание сценария
- preview сценария
- тип доступа сценария
- режим сценария

Поле `preview сценария` реализуется в виде формы загрузки изображения с устройства пользователя.

Поле `тип доступа сценария` содержит следующие значения:

- `публичный`
- `по ссылке`
- `черновик`

Поле `режим сценария` реализуется в виде переключателя и содержит следующие значения:

- `оцениваемый`
- `неоцениваемый`

Если сценарий переводится в режим `неоцениваемый`, то значения изменения счета у всех выборов должны быть очищены.

Во всплывающем окне доступны следующие кнопки:

- `Сохранить изменения`
- `Закрыть`

---

#### 13.6. Логика локального редактирования

Редактирование сценария на странице конструктора происходит в локальном состоянии страницы.

- изменения в полях шага применяются сразу к локальному состоянию
- изменения в полях выбора применяются сразу к локальному состоянию
- изменения в информации о сценарии применяются к локальному состоянию после подтверждения во всплывающем окне

При нажатии на кнопку `Сохранить изменения` текущее локальное состояние сценария должно сохраняться в постоянное хранилище.

После перезагрузки страницы без сохранения должна загружаться последняя сохраненная версия сценария.

---

#### 13.7. Логические ограничения

Для конструктора сценариев должны соблюдаться следующие ограничения:

- в сценарии всегда существует ровно один начальный шаг
- начальный шаг нельзя удалить
- тип начального шага нельзя изменить
- шаг типа `текст` может иметь только один следующий шаг
- шаг типа `конец` не имеет следующих шагов
- шаг типа `выбор` может содержать от 1 до 4 выборов
- поле изменения счета пользователя доступно только для оцениваемого сценария
